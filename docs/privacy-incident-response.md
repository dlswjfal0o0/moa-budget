# 개인정보 유출 대응 체크리스트

내부 운영자용 문서입니다. 앱에 노출되지 않으며, 개인정보보호법 제34조(개인정보 유출 등의 통지·신고)에 따른 대응 절차를 정리합니다.

## 신고 의무가 발생하는 경우

아래 중 하나에 해당하면 **인지 시점으로부터 72시간 이내**에 개인정보보호위원회(또는 한국인터넷진흥원)에 신고해야 합니다. 1인 사업자도 예외가 아닙니다.

- 1,000명 이상의 정보주체에 관한 개인정보가 유출된 경우
- 민감정보 또는 고유식별정보가 유출된 경우
- 해킹 등 외부의 불법적인 접근에 의해 개인정보가 유출된 경우

이 앱 기준으로 유출 시나리오가 될 수 있는 것: Firebase 프로젝트 관리자 계정 탈취, Firestore 보안 규칙 오설정으로 인한 전체 사용자 데이터 노출, `ANTHROPIC_API_KEY`/Firebase 서비스 계정 키 등 시크릿 유출로 인한 우회 접근, Vercel 환경변수 노출 등.

## 대응 절차

### 1. 인지 및 초기 파악 (인지 즉시)
- 유출 정황을 인지한 시각을 기록한다 (72시간 카운트다운의 기준점).
- Firebase Authentication / Firestore 콘솔에서 비정상 접근 로그, 대량 조회/다운로드 흔적을 확인한다.
- 가능하다면 즉시 노출된 크리덴셜(API 키, 서비스 계정 키)을 회수(rotate)하고, Firestore 보안 규칙을 임시로 더 제한적으로 조인다.

### 2. 범위 확정
- 영향받은 사용자 수, 유출된 개인정보 항목(이메일, 가계부 데이터, 카드/계좌 정보 등)을 특정한다.
- `firestore.rules`, Firebase 감사 로그, Vercel 배포/함수 로그를 근거로 유출 원인과 범위를 문서화한다.

### 3. 신고 (72시간 이내)
- **개인정보보호위원회**: privacy.go.kr 온라인 신고 또는 개인정보침해신고센터(국번없이 118)
- **한국인터넷진흥원(KISA)**: 118 상담센터, https://privacy.kisa.or.kr
- 신고 시 포함할 내용: 유출된 개인정보 항목, 유출 시점과 경위, 유출로 인해 발생 가능한 피해와 대응 방법, 담당자 연락처, 정보주체가 취할 수 있는 조치(비밀번호 변경 등).

### 4. 정보주체(이용자) 통지 (지체 없이, 원칙적으로 72시간 이내)
- 이메일 또는 앱 내 공지로 아래 내용을 통지한다:
  - 유출된 개인정보 항목
  - 유출 시점과 경위
  - 이용자가 취할 수 있는 조치(비밀번호 변경, 계정 모니터링 등)
  - 앱의 대응 조치 및 피해 구제 절차
  - 문의 가능한 연락처 (0o0moa030@gmail.com)
- 유출 규모가 1,000명 이상이면 통지 사실을 지체 없이 개인정보보호위원회 또는 전문기관에 신고해야 한다 (3번과 별개 의무 아님 — 3번 신고에 통지 여부를 포함해서 보고).

### 5. 재발 방지
- 유출 원인이 된 취약점을 수정한다 (예: `firestore.rules` 보완, 노출된 키 폐기 및 재발급, 접근 권한 최소화).
- 사고 경위, 조치 내역, 재발 방지 대책을 내부 기록으로 남긴다 (규제기관 요청 시 소명 자료로 사용).

## 평소 준비해 둘 것

- Firebase 프로젝트 소유자 계정에 2단계 인증 활성화
- `ANTHROPIC_API_KEY` 등 시크릿은 Vercel 환경변수로만 관리하고 저장소에 커밋하지 않는다 (현재 `.env.local`은 git에 커밋되지 않는지 주기적으로 확인)
- `firestore.rules`가 `users/{uid}`, `transactions`(uid 필드 기준)를 본인 계정으로만 제한하는지 정기적으로 재확인
- 이 체크리스트의 연락처(118, privacy.go.kr, privacy.kisa.or.kr)는 시간이 지나면 바뀔 수 있으니, 실제 사고 발생 시 최신 연락처를 다시 확인할 것
